# Contributing

Contributions are welcome if they improve clarity, usability, student safety, or alignment with recognized security guidance.

## Preferred contribution types

- Corrections to security guidance or source references.
- More usable templates for student hardware projects.
- Better examples for low/medium/high-risk classifications.
- Safer default configurations for internet-connected devices.
- Accessibility and web-publication improvements.

## Contribution expectations

- Cite authoritative sources for substantive security changes.
- Do not include real credentials, API keys, private IP ranges, internal network diagrams, unpublished sponsor details, or personal data.
- Keep examples generic and safe.
- Preserve the checklist's student-lab workflow and practical tone.

## Pull request checklist

- [ ] No secrets or private information included.
- [ ] New security claims cite a recognized source.
- [ ] Templates remain generic and reusable.
- [ ] Risky examples include safety warnings or are avoided.
- [ ] Markdown renders cleanly.
