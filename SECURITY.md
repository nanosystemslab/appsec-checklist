# Security Policy

## Reporting a vulnerability

Please do not open a public issue for suspected vulnerabilities in this checklist, templates, examples, or associated repository configuration.

Report security concerns to:

- Email: `nslab@hawaii.edu`
- GitHub private vulnerability reporting: enable this feature if available for the repository.

Please include:

- Affected file, template, or checklist section.
- Description of the issue.
- Potential impact.
- Suggested mitigation, if known.

## Scope

In scope:

- Checklist content in this repository.
- Template files in this repository.
- Example GitHub Actions, Dependabot, and repository security configuration.

Out of scope:

- Third-party services, tools, or standards linked from this repository.
- Denial-of-service testing against university or public infrastructure.
- Physical testing that creates safety risk without prior permission.

## Response expectations

We will acknowledge good-faith reports as promptly as feasible and coordinate remediation.

## Good-faith research

Good-faith reports that avoid privacy violations, service disruption, data destruction, and unsafe physical testing are welcome. Do not test against university networks, student systems, or lab equipment without explicit authorization.
