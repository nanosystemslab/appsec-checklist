# Open Hardware Application Security Audit Checklist

This site provides a practical application-security audit checklist for open hardware projects with internet connectivity.

- [Full checklist](../CHECKLIST.md)
- [Reference sources](REFERENCE_SOURCES.md)
- [GitHub upload guide](GITHUB_UPLOAD_GUIDE.md)
- [Future update prompt](FUTURE_UPDATE_PROMPT.md)

## Quick rule

> No identity, no encryption, no update path, no logs, no internet.

Use this checklist before student hardware projects connect to Wi-Fi, Ethernet, cloud services, MQTT brokers, web dashboards, remote-control interfaces, or university networks.
